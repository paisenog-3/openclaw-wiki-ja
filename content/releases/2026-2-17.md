## 詳細

2026年2月17日にリリースされたv2026.2.17では、Claude Sonnet 4.6の正式サポート、Slackのネイティブストリーミング、iOS共有拡張、サブエージェントの安定性向上など多数の改善が行われました。

## 新機能

### Claude Sonnet 4.6 対応

`anthropic/claude-sonnet-4-6` がエイリアス・デフォルト設定に対応しました。Sonnet 4.5しか公開されていない環境向けのフォワード互換フォールバックも含まれています。

設定例：
```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "anthropic/claude-sonnet-4-6"
      }
    }
  }
}
```

💡 **ポイント**: モデルエイリアス `sonnet` を Sonnet 4.6 に割り当てることで、サブエージェントも含めて一括管理できます。

### Anthropic 1M コンテキストベータ

Opus・Sonnetでオプトイン形式の100万トークンコンテキストに対応しました。モデル設定に `params.context1m: true` を追加することで有効化できます。

```json
{
  "models": {
    "providers": {
      "anthropic": {
        "models": [
          {
            "id": "claude-sonnet-4-6",
            "params": { "context1m": true }
          }
        ]
      }
    }
  }
}
```

{{< callout type="warning" >}}
1Mコンテキストはベータ機能です。対象モデルとAnthropicのベータプログラム参加が必要です。
{{< /callout >}}

### Slack ネイティブストリーミング

`chat.startStream` / `appendStream` / `stopStream` を使ったネイティブのシングルメッセージテキストストリーミングに対応。返信スレッドの整合性を保ちながら、リアルタイムで応答が流れるようになりました。デフォルトで有効、失敗時は通常配信にフォールバックします。

### Telegram インラインボタンスタイル

インラインボタンに `style` プロパティ（`primary`・`success`・`danger`）が追加されました。ボタンの見た目を用途に応じて使い分けられます。

### Telegram ユーザーリアクション通知

ユーザーがメッセージにリアクションしたイベントがシステムイベントとして受け取れるようになりました。`channels.telegram.reactionNotifications` で通知範囲を設定できます。

### `/subagents spawn` コマンド

チャットコマンドから直接サブエージェントを起動できる `/subagents spawn` が追加されました。決定論的なサブエージェント有効化が可能になります。

### Discord 機能強化

- **`/exec` コマンドオプション**: `host` / `security` / `ask` / `node` オプションのオートコンプリートと構造化入力が追加
- **再利用可能コンポーネント**: `components.reusable=true` でボタン・セレクト・フォームを複数回使用可能
- **ボタン別ユーザー制限**: `allowedUsers` でボタンを押せるユーザーをホワイトリスト制御

### iOS 機能強化

- **Share Extension**: URL・テキスト・画像をgatewayのエージェントリクエストとして直接転送できる共有拡張が追加
- **Talk Mode - Background Listening**: アプリをバックグラウンドにしてもTalkモードを維持するトグルが追加（デフォルトOFF・バッテリー配慮）
- **Voice Directive Hint トグル**: ElevenLabsの音声切り替え指示を無効化してトークン節約

### Cron 改善

- **Webhookデリバリー分離**: `delivery.mode = "webhook"` でannounceとは独立したWebhook配信が可能に
- **CLIスタガリング制御**: `openclaw cron add/edit --stagger <duration>` で定期実行のタイミングをずらせるように。`--exact` で即時実行も可能
- **実行ログにモデル使用量を記録**: cronジョブ実行ごとのトークン使用量が記録・集計できるように

### メモリ/検索 強化

FTSフォールバックとクエリ拡張が追加され、メモリ検索の精度が向上しました。

### その他

- **`llms.txt` 発見**: デフォルトで有効化。訪問サイトの `llms.txt` を自動参照
- **Docker**: `OPENCLAW_INSTALL_BROWSER` ビルド引数でChromium + Xvfbをプリインストール可能に
- **OpenAI Codex CLI 認証連携**: Codex CLI用のauth providerが統合

## バグ修正

### サブエージェント・ツール

**コンテキスト溢れ防止の強化**: モデル呼び出し前に蓄積されたツール結果コンテキストを事前チェックし、超過分を切り詰めてコンテキストウィンドウ溢れによるクラッシュを防ぎます。

**`read` の自動ページング**: `limit` を指定しない場合に自動でチャンクをページングするようになり、モデルの `contextWindow` に応じて読み込み量を調整します。大きいコンテキストのモデルでより多く読み込めます。

**コンパクション後のガイダンス改善**: `[compacted: tool output removed]` / `[truncated: output exceeded]` マーカーが現れた際に、全文 `cat` ではなく小さいチャンクで再読み込みするよう明示的なガイダンスが追加されました。

### Cron

- スピンループ防止: cronジョブがスケジュール秒内に完了した場合に次回実行時刻を正しく進め、最小リファイア間隔を強制
- `timeoutSeconds: 0` をタイムアウトなし（1秒にクランプしない）として正しく処理
- cronのannounce出力がメインセッションに正しくルーティングされるよう修正

### セキュリティ

- **Exec環境変数インジェクション修正 (OC-09)**: 環境変数インジェクションによる認証情報窃取パスを修正
- **`$include` パストラバーサル防止**: `$include` 解決をトップレベルコンフィグディレクトリに限定し、シンボリックリンク等によるトラバーサルを防止
- **`safeBins` のパスハイジャック防止**: `safeBins` バイナリが信頼済みbinディレクトリ（システムデフォルト + 起動時の `PATH`）から解決されることを要求し、PATH書き換えによるトロイバイナリのバイパスを防止

### Telegram

- DM音声メモのトランスクリプション対応
- ポールアクションのワイヤリング修正

### WebChat

- reply/audio ディレクティブタグがチャット出力に表示されないよう修正

### その他

- リプライスレッドのsticky性が改善: ストリーミング・分割チャンク全体でreplyToIdを保持
- macOS: Sparkle appcatsのバージョン修正（2026.2.15からのアップデートが再び配信されるように）
- iOSオンボーディング: QRファーストオンボーディングウィザードと設定コードディープリンクのサポート追加
